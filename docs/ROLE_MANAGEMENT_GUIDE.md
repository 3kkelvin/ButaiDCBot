# 📋 Discord 伺服器身分組維護與管理手冊

> **【重要聲明】** 
> 本手冊提供給**伺服器管理人員與營運團隊**查閱。
> 為確保機器人功能（如 `/role view_position` 公職列表、身分組自動修復）正常運作，請全體管理員務必遵守以下規範！

---

## 1. 🛑 裝飾身分組（分隔線）命名規範

伺服器中的「裝飾用身分組」（即用於分隔視覺區塊的身分組）有嚴格的命名規則：

- **格式要求**：身分組名稱中**必須同時包含英文中括號 `[` 與 `]`**。
- **正確範例**：
  - `===[管理身分組]===`
  - `----[公務身分組]----`
  - `[]`
- **錯誤範例**：
  - `--- 管理身分組 ---` *(缺少中括號，機器人會無法辨識為裝飾身分組)*
  - `【 公務身分組 】` *(使用全形括號，機器人無法辨識)*

⚠️ **警告**：請勿隨意刪除或修改裝飾身分組名稱中的 `[` 與 `]`。若格式不符，機器人在讀取公職名單或執行分隔線修正時將會產生誤判。

---

## 2. 📌 身分組排序與公職列表的連動關係

當管理員或成員在 Discord 輸入 `/role view_position` 時，機器人會自動產生**公職人員列表**。

名單的排列順序**完全與 Discord 伺服器設定中的身分組層級排序連動**：

### 排序規則說明：
1. **頂層固定身分組**：
   - 頂層會優先固定顯示 **服主** -> **大管理** -> **技術公務員**。
2. **管理身分組區段**：
   - 機器人會抓取 `[管理]` 裝飾身分組**正下方**，直到遇見下一個裝飾身分組之間的所有身分組。
   - 身分組在 Discord 列表排在越上面，`/role view_position` 顯示的順序就會越靠前。
3. **公務身分組區段**：
   - 機器人會抓取 `[公務]` 裝飾身分組**正下方**，直到遇見下一個裝飾身分組之間的所有身分組。
   - 排序同樣依據 Discord 伺服器內的高低順序呈現。

💡 **小技巧**：若想調整某個管理或公務身分組在列表上的顯示順序，只需在 Discord 伺服器設定的「身分組」頁面中，直接**上下拖曳調整該身分組的位置**即可，無需修改任何程式碼！

## 3. ⚠️ 核心身分組刪除防範條款

以下身分組與機器人後端設定檔（Config）已進行 **ID 強制綁定**：

- **服主身分組**
- **大管理身分組**
- **技術公務員身分組**
- **[管理] Tag 裝飾身分組**
- **[公務] Tag 裝飾身分組**

🚨 **嚴禁直接在 Discord 伺服器中刪除上述身分組！**

- **若直接刪除會發生什麼事？**
  - 機器人執行 `/role view_position` 時將找不到對應的 ID，導致指令報錯或公職名單缺失。
  - 身份組層級檢核與權限判斷功能將會異常。

- **如果真的需要更換身分組該怎麼辦？**
  1. 請**先聯繫技術人員**。
  2. 由技術人員在機器人設定檔（Config）中更新對應的新身分組 ID。
  3. 確認設定檔更新完畢後，方可刪除舊身分組。

## 4. ❓ 常見問題與自我檢查 (FAQ)

> **Q1：為什麼新增了公務身分組，卻沒有出現在 `/role view_position` 名單中？**
> **A1**：請檢查該新身分組在 Discord 身分組列表中，是否放在 `[公務]` 裝飾身分組的下方。如果放錯位置（例如放到其他裝飾身分組下方），機器人就無法自動歸類為公務身分組。

> **Q2：為什麼某個人明明有公職身分，名單上卻顯示「無」？**
> **A2**：請確認該成員是否為 Discord 機器人。機器人帳號預設會自動過濾，不會列入公職人員名單中。
